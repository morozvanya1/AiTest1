// 01 — производные

Производная — это «наклон» функции

Производная показывает, как быстро меняется функция при небольшом изменении её аргумента. Если f(x) = x², то производная f'(x) = 2x. При x=3 функция «растёт со скоростью 6».

// Зачем это нейросетям

Нейросеть имеет функцию потерь L (Loss), которая показывает, насколько плохо модель предсказывает. Нам нужно уменьшить L. Производная говорит: «сдвинь параметр в эту сторону — и L уменьшится».

Три правила, которые покроют 90% случаев

d/dx [xⁿ] = n·xⁿ⁻¹     d/dx [eˣ] = eˣ     d/dx [ln x] = 1/xСтепенное правило, производная экспоненты, производная логарифма
Chain Rule: d/dx [f(g(x))] = f'(g(x)) · g'(x)Правило цепочки — основа backpropagation. Производная сложной функции = произведение производных.

Правило цепочки — самое важное. Backpropagation — это буквально многократное применение chain rule через все слои сети. Мы реализуем его вручную на Дне 7.

// Числовая производная (для проверки)

Если не знаете аналитическую производную — считайте численно:
f'(x) ≈ (f(x+h) - f(x-h)) / (2h) при h=0.0001.
Это работает всегда и часто используется для проверки аналитических градиентов.

// 02 — градиентный спуск

Градиентный спуск

Представьте горный рельеф. Вы стоите в случайной точке и хотите спуститься в долину (минимум функции потерь). Вы смотрите вниз по склону и делаете шаг туда. Повторяете. Это и есть градиентный спуск.

w := w − α · ∂L/∂ww — параметр (вес), α — learning rate (размер шага), ∂L/∂w — градиент (производная Loss по w)

Знак «минус» означает, что мы идём против градиента — туда, где функция убывает. Если производная положительна (функция растёт) — уменьшаем w. Если отрицательна — увеличиваем.

Learning Rate — самый важный гиперпараметр

// Три сценария

α слишком мал (0.00001) → обучение займёт вечность
α слишком велик (10.0) → «прыгаем» мимо минимума, расходимся
α в самый раз (0.01–0.001) → плавно сходимся к минимуму

GPT-3 обучался с α≈3×10⁻⁴ со scheduler (постепенно уменьшался).

// 04 — практика

Первый обучаемый нейрон

Реализуем линейную регрессию через градиентный спуск. Одна входная переменная, один вес, одно смещение. Это простейшая «нейросеть» — 1 нейрон без функции активации.

Python · lesson_03.py
# Урок 3 — Градиентный спуск: обучаем первый нейрон import numpy as np # ───────────────────────────────────────────── # Данные: площадь квартиры → цена (условные ед.) # ───────────────────────────────────────────── X = np.array([30, 50, 70, 90, 110], dtype=float) y = np.array([150, 250, 330, 430, 520], dtype=float) # Нормализация (важно для стабильности обучения) X = (X - X.mean()) / X.std() y = (y - y.mean()) / y.std() # ───────────────────────────────────────────── # Модель: y_hat = w * x + b # Loss: MSE = mean((y_hat - y)²) # Градиенты аналитически: # ∂L/∂w = 2/N * Σ (y_hat - y) * x # ∂L/∂b = 2/N * Σ (y_hat - y) # ───────────────────────────────────────────── w, b = 0.0, 0.0 # инициализация нулями lr = 0.1 # learning rate epochs = 100 for epoch in range(epochs): # Forward pass y_hat = w * X + b # Loss (MSE) loss = ((y_hat - y) ** 2).mean() # Градиенты (производные Loss по w и b) dw = (2 / len(X)) * ((y_hat - y) * X).sum() db = (2 / len(X)) * (y_hat - y).sum() # Gradient descent step w -= lr * dw b -= lr * db if epoch % 10 == 0: print(f"Epoch {epoch:3d} | Loss: {loss:.6f} | w={w:.4f} b={b:.4f}") print(f"\nФинал: w={w:.4f}, b={b:.4f}") print(f"Предсказание для X=80м²: {w * ((80-70)/20.92) + b:.3f} (норм.)") # ───────────────────────────────────────────── # ПРОВЕРКА: численный градиент vs аналитический # Если разница мала — вы всё правильно сделали # ───────────────────────────────────────────── def loss_fn(w, b): return (((w * X + b) - y) ** 2).mean() h = 1e-5 dw_num = (loss_fn(w+h, b) - loss_fn(w-h, b)) / (2*h) dw_ana = (2/len(X)) * ((w*X+b-y)*X).sum() print(f"\nЧисленный градиент: {dw_num:.8f}") print(f"Аналитический градиент: {dw_ana:.8f}") print(f"Разница: {abs(dw_num - dw_ana):.2e} (должна быть ~1e-8)")
// Что значит каждая строка

y_hat = w*X + b — forward pass (предсказание)
loss = MSE — насколько мы ошиблись
dw, db — в какую сторону двигать веса
w -= lr * dw — один шаг обучения

В PyTorch это делает loss.backward() + optimizer.step() — автоматически.
