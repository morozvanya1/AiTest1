Вектор — это просто список чисел

В прошлом уроке мы создавали one-hot векторы. Вектор — это упорядоченный список чисел. В ML каждое слово, каждый пиксель, каждый признак хранится как вектор.

// Связь с Днём 1

Токен «кошка» → индекс 3 → one-hot вектор [0,0,0,1,0,...,0]. GPT-4 хранит каждый токен как вектор из 12288 чисел. Это и есть embedding — мы делали его вручную.

Скалярное произведение

Сумма произведений. Это СЕРДЦЕ attention-механизма. Результат — одно число.

📐

Норма (длина)

√(x₁²+x₂²+...) — «длина» вектора. Нужна для нормализации и косинусного сходства.

Скалярное произведение — интуиция

Скалярное произведение (dot product) двух векторов — это мера их «согласованности». Именно так Self-Attention решает: насколько «слово A» важно для «слова B»?

a · b = a₁·b₁ + a₂·b₂ + ... + aₙ·bₙ = |a| · |b| · cos(θ) Если векторы «смотрят в одну сторону» → большое число. Перпендикулярны → ноль. Противоположны → отрицательное.

Матрица — таблица векторов

Транспонирование — отражение матрицы

В формуле Attention: Attention = softmax(QKᵀ / √d) · V
Матрица Q имеет размер [seq_len × d_k], матрица K — тоже [seq_len × d_k]. Чтобы перемножить их, нужно транспонировать K → [d_k × seq_len]. Результат QKᵀ будет [seq_len × seq_len] — матрица сходств каждого токена с каждым.

// 03 — умножение матриц

MatMul — самая важная операция в ML

Умножение матриц (matrix multiplication, matmul) — ключевая операция любой нейросети. Это НЕ поэлементное умножение. Это особая операция: каждая ячейка результата — скалярное произведение строки первой матрицы на столбец второй.

C[i,j] = Σ A[i,k] · B[k,j] → (m×n) · (n×p) = (m×p) Внутренние размерности должны совпадать. Результат имеет внешние размерности.
// Правило совместимости размеров

(3×4) · (4×2) = (3×2) ✓  —  внутренняя 4 совпадает
(3×4) · (3×2) = ✗  —  4 ≠ 3, умножение невозможно
Это самая частая ошибка новичков в PyTorch. Запомните раз и навсегда.

Почему GPU так хорошо делает MatMul?

GPU имеет тысячи маленьких ядер, каждое может считать одно умножение независимо. Матрица 1000×1000 = 1 млн ячеек результата, каждая вычисляется параллельно. CPU делает это последовательно. Разница — в 100–1000 раз по скорости.

// Инсайт для понимания трансформера

Весь forward pass трансформера — это серия матричных умножений. X · W_q → матрица запросов Q. X · W_k → матрица ключей K. Q · Kᵀ → матрица сходств. Ваш SQL-навык поможет думать об этом как о JOIN-ах: каждая строка «ищет» похожие столбцы.

// 04 — практика

Пишем MatMul вручную — и сравниваем с NumPy

Цель: реализовать матричное умножение без np.dot и @, затем проверить результат через NumPy. Если результаты совпадают — вы поняли.

Python · lesson_02.py
# Урок 2 — Линейная алгебра для ML # pip install numpy (уже есть с Дня 1) import numpy as np # ══════════════════════════════════════════════ # ЧАСТЬ 1: Ручная реализация MatMul # Это поможет понять, что происходит "внутри" # ══════════════════════════════════════════════ def matmul_manual(A: list, B: list) -> list: """ Умножение матриц без NumPy. A: список списков размера [m × k] B: список списков размера [k × n] Возвращает: список списков размера [m × n] """ m, k = len(A), len(A[0]) k2, n = len(B), len(B[0]) assert k == k2, f"Несовместимые размеры: ({m}×{k}) · ({k2}×{n})" # Создаём результирующую матрицу m×n, заполненную нулями C = [[0.0] * n for _ in range(m)] for i in range(m): # строки A for j in range(n): # столбцы B for p in range(k):# скалярное произведение C[i][j] += A[i][p] * B[p][j] return C # ══════════════════════════════════════════════ # ЧАСТЬ 2: Transpose # ══════════════════════════════════════════════ def transpose(A: list) -> list: """Транспонирует матрицу A (m×n) → (n×m).""" m, n = len(A), len(A[0]) return [[A[i][j] for i in range(m)] for j in range(n)] # ══════════════════════════════════════════════ # ЧАСТЬ 3: Softmax — нужен уже сейчас # В Attention: softmax(QKᵀ / √d) # Превращает вектор чисел в вероятности (сумма=1) # ══════════════════════════════════════════════ def softmax(x: np.ndarray) -> np.ndarray: """ Numerically stable softmax. Вычитаем max для предотвращения переполнения — это стандартный трюк, используется везде. """ e = np.exp(x - np.max(x)) return e / e.sum() # ══════════════════════════════════════════════ # ЧАСТЬ 4: Мини-Attention (preview Дня 35!) # Упрощённый scaled dot-product attention # для 3 токенов и d_k = 3 # ══════════════════════════════════════════════ def scaled_dot_product_attention(Q, K, V): """ Q, K, V: матрицы [seq_len × d_k] Возвращает: взвешенные значения [seq_len × d_k] """ d_k = Q.shape[-1] # 1. Считаем сходства: (seq_len × seq_len) scores = Q @ K.T / np.sqrt(d_k) # 2. Softmax → веса внимания weights = np.array([softmax(row) for row in scores]) # 3. Взвешенная сумма значений return weights @ V, weights # ════════════════════════════════════════════ # ЗАПУСК И ПРОВЕРКА # ════════════════════════════════════════════ np.random.seed(42) # --- Тест matmul --- A = [[1, 2, 3], [4, 5, 6]] # 2×3 B = [[7, 8], [9, 10], [11, 12]] # 3×2 C_manual = matmul_manual(A, B) C_numpy = (np.array(A) @ np.array(B)).tolist() print("=== MatMul ===") print(f"Ручной: {C_manual}") # [[58, 64], [139, 154]] print(f"NumPy: {C_numpy}") print(f"Совпадают: {C_manual == C_numpy}\n") # --- Мини-Attention --- seq_len, d_k = 3, 4 # 3 токена, каждый представлен вектором 4D Q = np.random.randn(seq_len, d_k) K = np.random.randn(seq_len, d_k) V = np.random.randn(seq_len, d_k) output, attn_weights = scaled_dot_product_attention(Q, K, V) print("=== Mini Attention ===") print(f"Веса внимания (каждая строка = на что смотрит токен):") print(np.round(attn_weights, 3)) print(f"\nСумма весов каждой строки: {attn_weights.sum(axis=1).round(3)}") print(f"(должна быть [1. 1. 1.] — это вероятности)")
