// 01 — теория

Что на самом деле делает языковая модель?

GPT, Claude, LLaMA — все они решают одну задачу: предсказать следующий токен по предыдущим. Ничего более. Всё «понимание» и «рассуждение» — эмерджентное свойство этой простой цели, обученной на достаточно большом корпусе.

P(wt+1 | w1, w2, ..., wt) Вероятность следующего слова (токена) при условии всех предыдущих

Что такое токен?

Модели не работают с буквами или словами напрямую. Они работают с токенами — кусочками текста. Слово "привет" — один токен. Слово "трансформер" — может быть два: "транс" + "формер". Токенизатор GPT-4 разбивает любой текст на ~100 000 возможных токенов.

🔤

Токенизация

Текст → числа. BPE-алгоритм строит словарь из частых подстрок. "unhappiness" → ["un", "happiness"]

📐

Embedding

Каждый токен → вектор из 768–4096 чисел. Похожие слова — близкие векторы. Это «смысл» в числах.

🧠

Attention

Механизм, который решает: на какие предыдущие токены смотреть при предсказании следующего.

🎲

Sampling

Модель выдаёт распределение вероятностей. Temperature=0 → детерминировано. Temperature=1 → творчески.

Почему это «нейронная сеть»?

Внутри — миллиарды чисел (веса). При обучении мы подаём текст, модель делает предсказание, считаем ошибку, и чуть-чуть подталкиваем веса в правильную сторону. Делаем это миллиарды раз — веса «запоминают» язык.

// 02 — архитектура

Трансформер: что внутри?

Архитектура Transformer (Vaswani et al., 2017) — фундамент всех современных LLM. Упрощённо, токены проходят через стек блоков, каждый из которых состоит из двух частей: 

  INPUT TEXT: "Кошка сидит на"
       │
       ▼
  ┌─────────────────────────┐
  │    TOKENIZER (BPE)      │  "Кошка" → 1234, "сидит" → 567, "на" → 89
  └────────────┬────────────┘
               │
       ▼
  ┌─────────────────────────┐
  │  TOKEN EMBEDDING        │  [1234] → вектор 768 чисел
  │  POSITIONAL ENCODING    │  + «я стою на позиции 3»
  └────────────┬────────────┘
               │
       ▼
  ╔═════════════════════════╗  ┐
  ║  TRANSFORMER BLOCK × N  ║  │ Для GPT-2: N=12
  ║  ┌──────────────────┐   ║  │ Для GPT-4: N≈96
  ║  │ SELF-ATTENTION   │   ║  │
  ║  │  (кто на кого    │   ║  │ ← Самое важное!
  ║  │   смотрит)       │   ║  │
  ║  └────────┬─────────┘   ║  │
  ║  ┌────────▼─────────┐   ║  │
  ║  │  FEED-FORWARD    │   ║  │ Обычная нейросеть
  ║  │  NETWORK (FFN)   │   ║  │ 2 слоя, ReLU/GELU
  ║  └──────────────────┘   ║  │
  ╚═════════════════════════╝  ┘
               │
       ▼
  ┌─────────────────────────┐
  │  LINEAR + SOFTMAX       │  → вероятности по всему словарю
  └─────────────────────────┘
               │
       ▼
  OUTPUT: "коврике" (с вероятностью 0.42)

Self-Attention простыми словами

При обработке слова "банк" в предложении «Я пошёл в банк за деньгами» механизм внимания видит "деньгами" и понимает: речь о финансовом банке, не о берегу реки. Он динамически решает, какие токены важны для понимания текущего.

Attention(Q, K, V) = softmax( QKᵀ / √dk ) · V Q=запросы, K=ключи, V=значения — все получены умножением на обучаемые матрицы

Не пугайтесь формулы — мы разберём её пошагово на Дне 35. Сегодня важно понять идею: токены «общаются» между собой через умножение матриц.

// 03 — практика

Первый код: токенизация и векторы

Установите зависимость: pip install numpy. Сегодня мы реализуем упрощённую токенизацию и посмотрим, как слова превращаются в числа.

// Задача урока

Написать три функции: токенизатор на пробелах, one-hot encoding, и косинусное сходство — без библиотек кроме NumPy.

Python · lesson_01.py
# Урок 1 — Токены, векторы, сходство # Запустите: python lesson_01.py import numpy as np # ──────────────────────────────────────────────── # 1. Наивная токенизация (разбивка по пробелам) # В реальных LLM используется BPE — изучим позже # ──────────────────────────────────────────────── def build_vocab(corpus: list[str]) -> dict: """Строит словарь {слово: индекс} из списка предложений.""" words = set() for sentence in corpus: words.update(sentence.lower().split()) return {word: idx for idx, word in enumerate(sorted(words))} def tokenize(sentence: str, vocab: dict) -> list[int]: """Предложение → список индексов токенов.""" return [vocab.get(w, -1) for w in sentence.lower().split()] # ──────────────────────────────────────────────── # 2. One-Hot Encoding — простейший способ # превратить токен в вектор # (В LLM используются learned embeddings, но # one-hot — отправная точка понимания) # ──────────────────────────────────────────────── def one_hot(token_id: int, vocab_size: int) -> np.ndarray: """Токен → one-hot вектор.""" vec = np.zeros(vocab_size) vec[token_id] = 1.0 return vec # ──────────────────────────────────────────────── # 3. Косинусное сходство # Метрика «похожести» двух векторов # cos(θ) = 1 → одинаковые # cos(θ) = 0 → перпендикулярные (несвязанные) # cos(θ) = -1 → противоположные # ──────────────────────────────────────────────── def cosine_similarity(a: np.ndarray, b: np.ndarray) -> float: return np.dot(a, b) / (np.linalg.norm(a) * np.linalg.norm(b) + 1e-8) # ════════════════════════════════════════════════ # ЗАПУСК # ════════════════════════════════════════════════ corpus = [ "кошка сидит на ковре", "собака лежит на диване", "кошка и собака друзья", ] vocab = build_vocab(corpus) print(f"Словарь ({len(vocab)} слов): {vocab}\n") # Токенизируем первое предложение tokens = tokenize("кошка сидит на ковре", vocab) print(f"Токены: {tokens}") # Сравниваем «кошка» и «собака» vocab_size = len(vocab) vec_koshka = one_hot(vocab["кошка"], vocab_size) vec_sobaka = one_hot(vocab["собака"], vocab_size) vec_diwan = one_hot(vocab["диване"], vocab_size) sim_ks = cosine_similarity(vec_koshka, vec_sobaka) sim_kd = cosine_similarity(vec_koshka, vec_diwan) print(f"\nСходство кошка ↔ собака : {sim_ks:.3f}") # 0.000 (one-hot не знает) print(f"Сходство кошка ↔ диване : {sim_kd:.3f}") # 0.000 print(""" [!] Заметьте: one-hot не видит, что кошка и собака похожи (оба животные). Это проблема решается LEARNED EMBEDDINGS — изучим на Дне 16. """)
// Ожидаемый вывод

Все сходства будут 0.000 — и это правильно! One-hot векторы ортогональны. В реальных LLM у «кошки» и «собаки» косинусное сходство ~0.72. Это и есть смысл обучения эмбеддингов — тема Дня 16.
