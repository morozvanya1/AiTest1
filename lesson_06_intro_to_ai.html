// 01 — инициализация

Начальные веса — не «просто случайные»

Казалось бы: берём случайные числа и обучаем. Но масштаб этих чисел критически важен. Слишком большие → сигнал взрывается через слои. Слишком маленькие → сигнал затухает. Обе ситуации = сеть не обучается.

// Распределение активаций на глубокой сети (10 слоёв)

Xavier: W ~ N(0, √(2/(fan_in + fan_out)))Для tanh/sigmoid. Сохраняет дисперсию активаций одинаковой во всех слоях.
He (Kaiming): W ~ N(0, √(2/fan_in))Для ReLU. Учитывает, что ReLU «убивает» половину нейронов. Используется в ResNet, GPT.
// Инициализация нулями — особая ловушка

Если все веса = 0, все нейроны дают одинаковый выход → одинаковые градиенты → обучаются одинаково → нейроны «не специализируются». Это называется symmetry breaking problem. Случайность необходима!

// 02 — регуляризация

Переобучение и как его победить

Переобучение (overfitting) — модель «зазубрила» тренировочные данные, но не умеет обобщать. Нейросеть запомнила шум вместо паттернов. На тренировке Loss=0.01, на тесте Loss=5.0 — это переобучение.

// L2 Regularization (Weight Decay)

Добавляем к Loss штраф за большие веса: Loss_total = Loss + λ · Σ w². Это «давит» веса к нулю — модель не может «слишком сильно» запомнить любой конкретный признак. В AdamW (который использует LLaMA, GPT) это встроено как weight_decay.

// L1 Regularization

Штраф: λ · Σ |w|. Создаёт разреженность — многие веса становятся точно 0. Полезно для feature selection, но редко используется в нейросетях напрямую.

// 03 — dropout

Dropout — случайное «выключение» нейронов

Во время обучения каждый нейрон с вероятностью p (обычно 0.1–0.5) устанавливается в ноль. Сеть учится работать с «частичной информацией» → не может полагаться на конкретные нейроны → лучше обобщает.

Dropout(x, p): каждый xᵢ → 0 с вероятностью p, иначе xᵢ/(1-p) Деление на (1-p) — «inverted dropout»: сохраняет ожидаемое значение. При инференсе dropout отключается.
// Dropout в трансформерах

В современных LLM Dropout используется осторожно (p=0.1) или вообще не используется при очень больших данных. Для GPT-3 — минимально. Для дообучения на малых датасетах — полезен. В Claude и LLaMA применяется Weight Decay как основная регуляризация.

// 04 — практика

Сравниваем инициализации на практике

Python · lesson_06.py
# Урок 6 — Инициализация весов и Dropout import numpy as np np.random.seed(0) # ───────────────────────────────────────────── # Функции инициализации # ───────────────────────────────────────────── def init_zeros(shape): return np.zeros(shape) def init_large(shape): return np.random.randn(*shape) * 10.0 def init_xavier(shape): fan_in, fan_out = shape[0], shape[1] std = np.sqrt(2.0 / (fan_in + fan_out)) return np.random.randn(*shape) * std def init_he(shape): fan_in = shape[0] std = np.sqrt(2.0 / fan_in) return np.random.randn(*shape) * std # ───────────────────────────────────────────── # Симуляция forward pass через 10 слоёв # Смотрим на std активаций # ───────────────────────────────────────────── def simulate_forward(init_fn, n_layers=10, dim=256): x = np.random.randn(32, dim) stds = [x.std()] for _ in range(n_layers): W = init_fn((dim, dim)) x = np.maximum(0, x @ W) # ReLU stds.append(x.std()) return stds inits = { 'zeros': (init_zeros, "Нули — нет разнообразия"), 'large': (init_large, "×10 — взрыв активаций"), 'xavier': (init_xavier, "Xavier — стабильно"), 'he': (init_he, "He — лучший для ReLU"), } print("STD активаций по слоям (хочем ~1.0 везде):") for name, (fn, desc) in inits.items(): stds = simulate_forward(fn) stds_str = " → ".join(f"{s:.2f}" for s in stds[:5]) print(f"{name:8s}: {stds_str} | {desc}") # ───────────────────────────────────────────── # Dropout # ───────────────────────────────────────────── def dropout(x, p=0.3, training=True): """Inverted dropout. p — вероятность обнулить нейрон.""" if not training: return x # при инференсе не применяем mask = np.random.rand(*x.shape) > p return x * mask / (1.0 - p) # inverted: масштабируем обратно # Тест: среднее значение должно оставаться ~одинаковым x_test = np.ones((10000,)) print(f"\nDropout test (p=0.3):") print(f" До dropout: mean={x_test.mean():.3f}") print(f" После (train): mean={dropout(x_test, p=0.3, training=True).mean():.3f}") print(f" После (infer): mean={dropout(x_test, p=0.3, training=False).mean():.3f}") # ───────────────────────────────────────────── # L2 регуляризация в Loss # ───────────────────────────────────────────── def loss_with_l2(predictions, targets, weights_list, lam=0.01): mse = ((predictions - targets) ** 2).mean() l2 = sum((w**2).sum() for w in weights_list) return mse + lam * l2 W1 = init_he((4, 8)); W2 = init_he((8, 1)) pred = np.random.rand(10); tgt = np.random.rand(10) print(f"\nL2 regularization:") print(f" MSE only: {((pred-tgt)**2).mean():.4f}") print(f" MSE + L2(0.01): {loss_with_l2(pred, tgt, [W1, W2], 0.01):.4f}")
// Ожидаемый результат

Xavier и He должны показать STD ≈1.0 через все слои. Zeros — STD=0.0 сразу. Large — огромные числа, потом NaN. Это объясняет, почему правильная инициализация = обязательное требование, не опция.
