// 01 — вероятность

Softmax: числа → вероятности

Языковая модель на выходе даёт вектор из ~50 000 чисел — по одному на каждый токен словаря. Эти числа (логиты) надо превратить в вероятности. Для этого используется softmax.

softmax(zᵢ) = e^zᵢ / Σⱼ e^zʲЭкспонента усиливает разницу между числами. Деление нормирует: все вероятности суммируются в 1.

// Temperature в LLM

Когда вы ставите temperature=0 в Claude — это делит логиты на очень маленькое число перед softmax. Самый вероятный токен получает вероятность ≈1, остальные ≈0. При temperature=1 — стандартный softmax. При temperature=2 — распределение «сглаживается», модель «рандомнее».

// 02 — энтропия

Информационная энтропия

Энтропия измеряет неопределённость распределения. Чем равномернее распределение — тем выше энтропия (больше неопределённости). Чем сконцентрированнее — тем ниже.

H(p) = −Σᵢ p(i) · log₂ p(i)Измеряется в битах. Равномерное распределение из N вариантов: H = log₂(N). Точное предсказание: H = 0.

В контексте LLM: если модель уверена в следующем слове (энтропия низкая) — она «хорошо обучена» на этом контексте. Высокая энтропия означает «я не знаю, что дальше».

// Связь с perplexity

Perplexity = 2^H — стандартная метрика качества LLM. GPT-2 имеет perplexity ~18 на WikiText. GPT-4 — значительно ниже. Чем меньше — тем лучше модель «понимает» язык. Эту метрику вы увидите в каждом paper по LLM.

// 03 — функции потерь

Кросс-энтропия — сердце LLM

Языковые модели обучаются минимизировать кросс-энтропийную потерю. Для каждого токена в тексте модель должна угадать следующий — и за ошибку «штрафуется» пропорционально своей уверенности.

CE(y, ŷ) = −log(ŷ[correct_token])Если модель дала правильному токену вероятность 0.9 → Loss = −log(0.9) = 0.105. Если 0.01 → Loss = 4.6. Чем выше уверенность — тем меньше штраф.
Функция потерьКогда использоватьВ LLM?
MSE (L2)Регрессия — предсказание числаНет
MAE (L1)Регрессия, устойчивая к выбросамНет
Cross-EntropyКлассификация, предсказание токеновДа ✓
KL-DivergenceСравнение распределений (RLHF)Да ✓
// Почему не MSE для языка?

Токены — это категории, не числа. Расстояние между токеном 1234 («кошка») и 1235 («собака») бессмысленно. MSE предполагает, что «ошибиться на 1» лучше, чем «ошибиться на 100» — но для токенов это неверно. Кросс-энтропия работает с вероятностями и не предполагает порядка.

// 04 — практика

Реализуем Loss для языковой модели

# Урок 4 — Вероятность и функции потерь для LLM import numpy as np # ───────────────────────────────────────────── # 1. Softmax (стабильная версия) # ───────────────────────────────────────────── def softmax(logits): logits = logits - logits.max() # stability trick exp = np.exp(logits) return exp / exp.sum() # ───────────────────────────────────────────── # 2. Кросс-энтропийная потеря (один токен) # ───────────────────────────────────────────── def cross_entropy(probs, target_idx): """ probs: вектор вероятностей [vocab_size] target_idx: индекс правильного токена """ return -np.log(probs[target_idx] + 1e-12) # ───────────────────────────────────────────── # 3. Loss на батче (несколько токенов) # ───────────────────────────────────────────── def batch_cross_entropy(logits_batch, targets): """ logits_batch: [batch_size × vocab_size] targets: [batch_size] — индексы правильных токенов """ losses = [] for logits, target in zip(logits_batch, targets): probs = softmax(logits) losses.append(cross_entropy(probs, target)) return np.mean(losses) # ───────────────────────────────────────────── # 4. Энтропия и Perplexity # ───────────────────────────────────────────── def entropy(probs): probs = probs[probs > 0] return -np.sum(probs * np.log2(probs)) def perplexity(loss_nats): """loss в натуральных логарифмах → perplexity""" return np.exp(loss_nats) # ════════════════════════════════════════════ # СИМУЛЯЦИЯ: представим маленький словарь из 5 токенов # ════════════════════════════════════════════ vocab = ["кошка", "собака", "сидит", "лежит", "на"] # Контекст: "кошка ___ на" # Правильный ответ: "сидит" (индекс 2) # Плохая модель (почти равномерное распределение) bad_logits = np.array([0.2, 0.1, 0.3, 0.1, 0.2]) # Хорошая модель (уверена в "сидит") good_logits = np.array([0.1, -1.0, 3.5, -0.5, 0.2]) target = 2 # "сидит" bad_probs = softmax(bad_logits) good_probs = softmax(good_logits) print("=== Плохая модель ===") for w, p in zip(vocab, bad_probs): print(f" {w:8s}: {p:.3f} {'←' if w=='сидит' else ''}") print(f" Loss: {cross_entropy(bad_probs, target):.4f}") print(f" Perplexity: {perplexity(cross_entropy(bad_probs, target)):.2f}\n") print("=== Хорошая модель ===") for w, p in zip(vocab, good_probs): print(f" {w:8s}: {p:.3f} {'←' if w=='сидит' else ''}") print(f" Loss: {cross_entropy(good_probs, target):.4f}") print(f" Perplexity: {perplexity(cross_entropy(good_probs, target)):.2f}\n") # Батч из 3 позиций текста print("=== Батч: Loss по всему предложению ===") batch_logits = np.random.randn(3, 5) # 3 позиции, словарь 5 batch_targets = np.array([0, 2, 4]) batch_loss = batch_cross_entropy(batch_logits, batch_targets) print(f" Средний Loss на батче: {batch_loss:.4f}") print(f" Perplexity батча: {perplexity(batch_loss):.2f}")
